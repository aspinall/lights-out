# Security

## Baseline

- Identify trust boundaries and threat actors in design records.
- Apply least privilege to people, agents, services, and CI.
- Validate authorization at the resource boundary, not only in the interface.
- Store secrets in an approved secret manager and rotate exposed credentials immediately.
- Pin and review dependencies appropriate to their supply-chain risk.
- Minimize collection and retention of personal or sensitive data.

## Data classification

Define project classifications, handling rules, retention, and deletion behavior here before production data is used.

## Reporting

Document the private vulnerability-reporting channel before public launch. Do not place exploitable vulnerability details in public issues.
