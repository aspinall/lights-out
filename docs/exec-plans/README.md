# Execution plans

Use a lightweight in-chat plan for small, obvious changes. Use a checked-in execution plan when work spans components, carries meaningful risk, requires research, or may continue across agent sessions.

1. Copy `PLAN_TEMPLATE.md` to `active/YYYY-MM-DD-short-name.md`.
2. Keep progress, discoveries, and decisions current while working.
3. Make acceptance criteria executable or directly observable.
4. On completion, write the outcome and move the file to `completed/`.
5. Record intentionally deferred work in `tech-debt.md`.

Plans are resumable working documents, not approval theater. A new agent should be able to continue using the plan and repository alone.
