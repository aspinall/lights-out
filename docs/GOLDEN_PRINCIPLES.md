# Golden principles

These are durable, reviewable defaults. Promote a principle into an automated check when violations recur.

1. Repository truth beats conversational context.
2. Validate at boundaries; never probe unknown data until something works.
3. Centralize an invariant instead of copying its implementation.
4. Prefer explicit domain language over generic abstractions.
5. Tests assert behavior and contracts, not implementation trivia.
6. Every failure tells the operator what happened and what to do next.
7. Observability is part of the feature, not follow-up work.
8. Delete obsolete paths when a migration is complete.
9. Small, reversible changes are easier to validate and repair.
10. A recurring mistake is a harness bug.
