# Design

## Principles

- Make the next action obvious.
- Preserve user work across errors and retries.
- Use consistent language for the same concept everywhere.
- Provide visible system status and actionable recovery guidance.
- Meet WCAG 2.2 AA for user-facing interfaces unless a documented constraint prevents it.

## Verification

For interface changes, capture the relevant states: loading, empty, error, partial, success, and restricted access. Exercise the real interaction path when tooling permits; static review alone is insufficient for behavioral changes.

Document product-specific tokens, components, voice, and supported viewports as the design system develops.
