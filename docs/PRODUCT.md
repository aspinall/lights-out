# Product

## Users and outcome

Replace this section with the primary users, the job they are trying to complete, and the observable outcome the product creates.

## Core beliefs

- Optimize for a complete user outcome, not feature count.
- State acceptance criteria in behavior users can observe.
- Prefer the smallest coherent change that tests the riskiest assumption.
- Treat accessibility, privacy, performance, and failure recovery as product behavior.

## Success measures

Define a small set of measurable leading and lagging indicators. Every metric must name its owner, data source, and decision it informs.
